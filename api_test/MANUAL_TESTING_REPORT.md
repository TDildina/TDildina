## Примеры найденных наблюдений
1. **Наблюдение:** При запросе несуществующего поста (`/posts/9999`) API возвращает статус 200 и пустой объект `{}` вместо 404.
>**Вывод:** Поведение соответствует работе mock-сервиса, но в реальном проекте ожидался бы 404. 

>Для тестирования реального API:
Добавить проверки статус кодов негативные сценарии должны возвращать 4xx/5xx.

2. **Наблюдение:** 2. При POST без поля `title` API принимает запрос и возвращает объект с пустой строкой в `title`.
>**Вывод:** Валидация на стороне клиента обязательна, сервер не проверяет обязательные поля.

>Для тестирования реального API: Реализовать авторизацию - проверка прав доступа к ресурсам.

3. **Наблюдение:** 3. При SQL инъекциB  в теле запроса API принимает запрос и возвращает 201

>**Вывод:** Поведение соответствует работе mock-сервиса, но в реальном проекте ожидался бы 404. 

>Для тестирования реального API: Добавить защиту от инъекций
